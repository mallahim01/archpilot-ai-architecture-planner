- The system will use a shared API and types across web and mobile applications to ensure consistency.
- The React Native app will consume the same API as the web application, allowing for shared TypeScript types/components.
- High-level screens/pages include: Public (Landing, course catalog/browse, course detail, pricing), Auth (signup/login, verify email, forgot password), Student (dashboard, course player, certificates, billing/subscription), Instructor (dashboard, course builder/manage, analytics, payout history), Admin (manage users/roles, courses/moderation, subscription plans, payouts, reports).
