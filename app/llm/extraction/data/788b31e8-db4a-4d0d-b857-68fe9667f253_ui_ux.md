- Web-only MVP confirmed for the initial launch.
- UI will proceed in phases over 4 months for development.
- Web-first product design.
- Roles for MVP: Admin, Student, Instructor (Teacher).
- Backend-relevant screens: Public (landing, course catalog/preview, pricing, checkout), Auth (sign up/in, forgot password), Student (dashboard, course player, progress, certificates, billing/subscription management), Instructor (instructor dashboard, course builder/manager, enrollment stats, payout/earnings view), Admin (user/instructor management, course moderation, subscription plan config, payout cycle management, reporting).
- Subscriptions unlock either all courses on the platform or a defined catalog.
- UI development to shape purchase flows and teacher earnings dashboards.
