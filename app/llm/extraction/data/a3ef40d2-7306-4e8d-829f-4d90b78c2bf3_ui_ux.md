- User prefers a clean, modern, professional interface with a light theme and neutral or premium accent colors.
- User emphasizes a dashboard-centric experience for all roles with clear separation between admin, teacher, and student views.
- UI should have minimal onboarding friction and be mobile-responsive and tablet-friendly.
- Design is intended for scalability and feature expansion.
- High-level screens identified for launch: Public (landing, course catalog, course detail, teacher profile, certificate verification), Auth (sign up/in, email/phone verification, password reset, optional SSO), Student (onboarding, subscription management, checkout, purchases/history, learning player, dashboard, certificates), Teacher (verification/KYC, course builder, media uploads, pricing, earnings/payouts, analytics), Admin (user/role management, teacher approval, course moderation, pricing rules, payout controls, certificate templates/rules, audit logs, platform analytics).
- The platform will be web-first and dashboard-style.
